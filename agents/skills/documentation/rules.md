# Documentation Rules

- **Use Markdown Standards**: Ensure all headings (`#`, `##`, `###`) are properly structured.
- **Provide Actionable Examples**: Do not use placeholders or hypothetical code snippet comments. Show complete, working examples.
- **Expose Diagrams**: Use Mermaid.js syntax for visual system diagrams (flowcharts, sequence diagrams, architecture diagrams).
- **Self-explanatory**: Ensure README files enable a new developer to get the application running in under 5 minutes.
