# GitHub Workflow Rules

- **Use Conventional Commits**: Commits must start with `feat:`, `fix:`, `docs:`, `refactor:`, `test:`, or `chore:` followed by a lowercase description.
- **Never Commit to Main**: All work must be conducted on feature/issue-specific branches. Merge using Pull Requests.
- **Descriptive PRs**: PR descriptions must detail what changed, how it was tested, and link to associated issues.
- **Clean Commits**: Avoid bloated commits. Group changes logically and run linter/tests before committing.
